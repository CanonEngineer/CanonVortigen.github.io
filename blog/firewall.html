O que é um Firewall?

Um firewall (parede de fogo) é um dispositivo ou software que monitora e controla o tráfego de rede com base em regras de segurança predefinidas. Funciona como um porteiro digital — decide o que entra, o que sai e o que é bloqueado.

Conceito-chave: O firewall opera nas camadas 3 (Rede) a 7 (Aplicação) do modelo OSI, dependendo do tipo.

Tipos de Firewall

Packet Filtering

Analisa cabeçalhos IP (origem, destino, porta, protocolo). Rápido, mas não vê conteúdo. Camada 3–4.

Stateful (Com Estado)

Rastreia conexões ativas. Sabe que um pacote de retorno pertence a uma sessão legítima. Padrão atual.

Application Layer (Proxy)

Inspeciona conteúdo das aplicações (HTTP, FTP, DNS). Mais lento, mas muito mais seguro. Camada 7.

NGFW (Next-Gen)

Combina firewall tradicional + IPS + filtro de aplicações + antivírus + SSL inspection. Palo Alto, Fortinet.

WAF (Web App Firewall)

Protege aplicações web contra SQL Injection, XSS e bots. Cloudflare, AWS WAF, ModSecurity.

Host-based (HFW)

Software instalado na máquina (Windows Firewall, iptables/nftables no Linux). Protege o host individual.

Regras e políticas de firewall

Toda configuração de firewall segue uma política de segurança. A regra de ouro é:

Deny by Default — bloqueie tudo por padrão e libere apenas o tráfego explicitamente necessário (princípio do menor privilégio).

Componentes de uma regra

  • Origem (Source): IP ou rede de onde vem o tráfego.
  • Destino (Destination): IP ou rede para onde vai.
  • Porta/Protocolo: TCP/UDP, número da porta (80, 443, 22...).
  • Ação: ALLOW (permitir), DENY (bloquear), LOG (registrar).
  • Direção: Inbound (entrada), Outbound (saída).
# Exemplo: iptables (Linux) # Bloquear tudo por padrão iptables -P INPUT DROP iptables -P FORWARD DROP # Permitir SSH apenas da rede admin iptables -A INPUT -p tcp -s 192.168.10.0/24 --dport 22 -j ACCEPT # Permitir HTTP/HTTPS de qualquer origem iptables -A INPUT -p tcp --dport 80 -j ACCEPT iptables -A INPUT -p tcp --dport 443 -j ACCEPT # Permitir tráfego de retorno (established) iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

Zonas de segurança (Security Zones)

Firewalls modernos organizam interfaces em zonas com níveis de confiança diferentes:

  • WAN: internet — zona não confiável, bloqueio máximo.
  • DMZ: servidores acessíveis externamente, isolados da LAN.
  • LAN: rede interna — acesso restrito vindo de WAN/DMZ.
  • VPN Zone: usuários remotos autenticados com regras específicas.

Ferramentas populares

pfSense / OPNsense

Firewalls open-source baseados em FreeBSD. Muito usados em PMEs e labs. Interface web completa.

Fortinet FortiGate

NGFW líder de mercado. UTM completo com VPN, IPS, antivírus e SD-WAN integrados.

Palo Alto Networks

NGFW enterprise com App-ID, identificação de aplicações independente de porta.

iptables / nftables

Firewall nativo do Linux. Poderoso, configurável via CLI. Base de muitas soluções.
