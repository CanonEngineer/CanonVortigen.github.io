O que é virtualização?

Virtualização permite executar múltiplas máquinas virtuais (VMs) em um único servidor físico. Cada VM tem seu próprio sistema operacional, isolado das demais, compartilhando apenas o hardware subjacente através de um hypervisor.

Benefício principal: Melhor aproveitamento de hardware — um servidor que usaria 15% de capacidade pode hospedar dezenas de VMs.

Tipos de hypervisor

Tipo 1 (Bare-Metal)

Instalado diretamente no hardware. Maior performance. Proxmox, VMware ESXi, Hyper-V, Xen.

Tipo 2 (Hosted)

Roda sobre um SO hospedeiro. VirtualBox, VMware Workstation. Ideal para testes e labs.

Proxmox VE — Visão geral

Proxmox Virtual Environment (VE) é uma plataforma open-source de virtualização enterprise baseada em Debian Linux. Combina KVM para VMs completas e LXC para containers leves, tudo gerenciado por uma interface web intuitiva.

Principais recursos do Proxmox

  • Interface Web: gestão completa via browser na porta 8006 (HTTPS).
  • KVM/QEMU: VMs com qualquer SO — Linux, Windows, BSD.
  • LXC Containers: containers Linux nativos, mais leves que VMs.
  • Cluster HA: múltiplos nodes com failover automático.
  • Live Migration: mover VMs entre hosts sem downtime.
  • Backup/Restore: agendamento de backups com deduplicação.
  • Firewall integrado: regras por VM/container.
  • ZFS e Ceph: storage avançado com snapshots e replicação.

Regra prática: Use LXC para serviços Linux leves (Nginx, PostgreSQL, Redis). Use VM quando precisar de Windows ou isolamento total do kernel.

Instalação e configuração do Proxmox

Requisitos mínimos

  • CPU com virtualização (Intel VT-x / AMD-V) habilitada na BIOS.
  • 64-bit, 2+ cores (recomendado 4+).
  • RAM: mínimo 4GB (recomendado 16GB+ para produção).
  • Disco: 32GB+ (SSD fortemente recomendado).
  • Rede: 1+ interfaces (2 para bond/redundância).
# Após instalar Proxmox VE, acessar via browser: https://IP-DO-SERVIDOR:8006 # Login padrão: Usuário: root Senha: (definida na instalação) Realm: Linux PAM # Criar VM via CLI (qm): qm create 100 --name ubuntu-server --memory 2048 --cores 2 \ --net0 virtio,bridge=vmbr0 --scsi0 local-lvm:20 # Criar container LXC: pct create 101 local:vztmpl/ubuntu-22.04.tar.zst \ --hostname web-server --memory 512 --cores 1 \ --net0 name=eth0,bridge=vmbr0,ip=dhcp

VMs / LXC
  • vmbr0: bridge padrão — VMs ficam na mesma rede que o host.
  • VLAN aware: bridge pode transportar múltiplas VLANs (802.1Q).
  • SDN (Software Defined Networking): redes virtuais isoladas entre VMs.
  • Bonding: agregar múltiplas NICs para redundância (LACP).

Storage: ZFS e Ceph

local-lvm (LVM)

Storage padrão em disco local. Simples, rápido. Sem redundância entre nodes.

ZFS

Filesystem avançado com snapshots, compressão, deduplicação e integridade de dados (checksums).

Ceph

Storage distribuído para clusters. Replica dados entre nodes. Escalável para petabytes.

NFS / iSCSI

Storage compartilhado via rede. Útil para live migration e backups centralizados.

# Snapshot ZFS de uma VM (via Proxmox) qm snapshot 100 backup-antes-update --description "Antes do apt upgrade" # Rollback se algo der errado: qm rollback 100 backup-antes-update

Cluster e Alta Disponibilidade (HA)

Com 3+ nodes, o Proxmox forma um cluster usando Corosync para comunicação e quorum para decisões. Se um node cair, as VMs marcadas como HA reiniciam automaticamente em outro node.

# Criar cluster (no primeiro node): pvecm create cluster-homelab # Adicionar nodes ao cluster: pvecm add IP-DO-NODE-1 # Habilitar HA para uma VM: ha-manager add vm:100 --state started --max_restart 3

Backup e recuperação

  • vzdump: ferramenta nativa de backup para VMs e containers.
  • Modos: snapshot (sem downtime), suspend ou stop.
  • Formatos: tar.gz, zstd (compressão rápida), PBS (Proxmox Backup Server).
  • Agendamento: configure jobs diários/semanais na interface web.
  • Proxmox Backup Server (PBS): solução dedicada com deduplicação global.

Regra 3-2-1: 3 cópias dos dados, em 2 mídias diferentes, com 1 cópia offsite. Teste restaurações regularmente!
